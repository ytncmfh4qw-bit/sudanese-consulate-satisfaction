# منصة قياس رضا المواطن السوداني — خدمات القنصلية بجازان

هذا المشروع هو تطبيق ويب ثابت **Static Progressive Web App** باللغة العربية واتجاه **RTL**، مخصص لقياس رضا المواطنين السودانيين عن خدمة وصول القنصلية السودانية من جدة إلى منطقة جازان لإصدار وتجديد الجوازات. تعتمد النسخة الحالية على صفحة مواطن عامة لإرسال التقييمات، ولوحة إدارة محمية بتسجيل دخول حقيقي عبر **Supabase Auth**، مع سياسات **Row Level Security** تمنع قراءة النتائج أو حذفها إلا من حسابات الإدارة المصرح بها.[1] [2]

> في النسخة الآمنة، لا توجد كلمة مرور إدارة مخزنة داخل ملفات الواجهة. صفحة المواطن عامة، أما لوحة الإدارة فتتطلب حساب مستخدم في Supabase Auth، ويجب أن يكون بريد هذا الحساب موجوداً في جدول `survey_admins` داخل قاعدة البيانات.

## طريقة تهيئة Supabase الآمنة

افتح مشروعك في Supabase، ثم انتقل إلى **SQL Editor**، وأنشئ استعلاماً جديداً، ثم انسخ محتوى الملف `database.sql` كاملاً وشغّله. سيقوم الملف بإنشاء جدول التقييمات وجدول المديرين، وتفعيل **Row Level Security**، وإضافة سياسة تسمح للمواطن بإدخال تقييم فقط، وسياسات تسمح للمديرين المصادقين فقط بقراءة النتائج وحذفها.[1]

بعد تنفيذ الملف، انتقل إلى **Authentication > Users** داخل Supabase وأنشئ مستخدماً جديداً للمدير. استخدم بريداً حقيقياً وكلمة مرور قوية. بعد إنشاء المستخدم، ارجع إلى **SQL Editor** ونفّذ الأمر التالي بعد تغيير البريد إلى بريد المدير الحقيقي:

```sql
insert into public.survey_admins (email, full_name)
values ('your-admin-email@example.com', 'مدير النظام')
on conflict (email) do update set full_name = excluded.full_name;
```

يحتوي ملف `database.sql` على بريد افتراضي `admin@example.com` كنموذج توضيحي فقط. يجب استبداله ببريد المدير الحقيقي قبل الاعتماد النهائي. إذا دخل مستخدم بحساب صحيح في Supabase Auth لكن بريده غير موجود في `survey_admins`، فلن يستطيع قراءة النتائج بسبب سياسات RLS.

## طريقة التشغيل محلياً

يمكن فتح `index.html` مباشرة في المتصفح، لكن الأفضل تشغيل خادم محلي بسيط حتى تعمل الملفات كما ستعمل على الاستضافة. من داخل مجلد المشروع يمكن تنفيذ الأمر التالي:

```bash
python3 -m http.server 8000
```

بعدها افتح الرابط التالي في المتصفح:

```text
http://localhost:8000/index.html
```

ولوحة الإدارة ستكون على الرابط:

```text
http://localhost:8000/admin.html
```

## طريقة رفع الملفات على GitHub Pages

أنشئ مستودعاً جديداً على GitHub، ثم ارفع جميع ملفات المشروع إلى الفرع الرئيسي. بعد ذلك افتح إعدادات المستودع، وانتقل إلى قسم **Pages**، واختر النشر من الفرع الرئيسي ومن جذر المشروع. بعد الحفظ، سيمنحك GitHub رابطاً عاماً يشبه الآتي:[4]

```text
https://username.github.io/repository-name/
```

ستكون صفحة الاستبيان عادة على الرابط الأساسي نفسه، أما لوحة الإدارة فتكون على:

```text
https://username.github.io/repository-name/admin.html
```

نشر ملفات الواجهة كموقع عام أصبح مقبولاً في هذه النسخة لأن كلمة مرور الإدارة لم تعد موجودة داخل الكود. مع ذلك، يبقى مفتاح Supabase المنشور ظاهراً في الواجهة، وهذا طبيعي في تطبيقات الويب العامة بشرط أن تكون سياسات RLS مضبوطة كما في `database.sql`.[1] [2]

## طريقة إنشاء QR Code لرابط الاستبيان

بعد نشر المشروع على GitHub Pages، انسخ رابط صفحة الاستبيان العامة. يمكن إنشاء رمز QR من المتصفح أو من خدمة موثوقة، ثم وضع الرمز في موقع واضح داخل مقر الخدمة مع عبارة مختصرة مثل: **شاركنا تقييمك لتحسين الخدمة**.

## إدارة حسابات المديرين

إضافة مدير جديد لا تتطلب تعديل ملفات الموقع. أنشئ مستخدماً جديداً من **Authentication > Users** في Supabase، ثم أضف بريده إلى جدول `survey_admins` بالأمر التالي:

```sql
insert into public.survey_admins (email, full_name)
values ('new-admin@example.com', 'اسم المدير')
on conflict (email) do update set full_name = excluded.full_name;
```

لحذف صلاحية مدير، احذف بريده من الجدول:

```sql
delete from public.survey_admins
where email = 'old-admin@example.com';
```

تغيير كلمة مرور المدير يتم من لوحة Supabase ضمن إعدادات المستخدم في **Authentication**، وليس من ملفات المشروع. هذا هو الفرق الأساسي بين الحماية السابقة والحماية الحالية.

## بنية قاعدة البيانات

| الجدول | الوصف |
|---|---|
| `citizen_satisfaction_surveys` | يحتفظ بكل تقييمات المواطنين والحقول التحليلية اللازمة للوحة الإدارة. |
| `survey_admins` | يحتفظ بقائمة البريد الإلكتروني للحسابات المسموح لها بقراءة النتائج وإدارتها. |

| العمود في جدول التقييمات | النوع | الوصف |
|---|---|---|
| `id` | `uuid` | معرف داخلي فريد لكل تقييم. |
| `reference_no` | `text` | رقم مرجعي ظاهر للمواطن بعد الإرسال. |
| `service_type` | `text` | نوع الخدمة التي تم تقييمها. |
| `overall_rating` | `int` | التقييم العام من 1 إلى 5. |
| `speed_rating` | `int` | تقييم سرعة إنجاز الخدمة. |
| `staff_rating` | `int` | تقييم تعامل الموظفين. |
| `clarity_rating` | `int` | تقييم وضوح الإجراءات. |
| `organization_rating` | `int` | تقييم تنظيم الحضور والانتظار. |
| `communication_rating` | `int` | تقييم جودة التواصل قبل الحضور. |
| `completed_status` | `text` | حالة إنجاز المعاملة: نعم، جزئياً، لا. |
| `waiting_time` | `text` | مدة الانتظار المختارة. |
| `notes` | `text` | ملاحظات أو اقتراحات اختيارية. |
| `phone` | `text` | رقم الجوال اختياري. |
| `user_agent` | `text` | معلومات تقنية عامة عن المتصفح. |
| `created_at` | `timestamp with time zone` | وقت إنشاء التقييم تلقائياً. |

## اختبار سريع بعد التركيب

| خطوة الاختبار | النتيجة المتوقعة |
|---|---|
| فتح `index.html` من الجوال | تظهر واجهة عربية RTL متجاوبة وبأزرار كبيرة. |
| محاولة الإرسال دون تقييم عام | تظهر رسالة تطلب اختيار التقييم العام. |
| إرسال تقييم مكتمل | تظهر رسالة شكر ورقم مرجعي للتقييم. |
| تكرار الإرسال فوراً من نفس الجهاز | تظهر رسالة منع التكرار السريع. |
| فتح `admin.html` دون تسجيل دخول | تظهر شاشة الدخول بالبريد الإلكتروني وكلمة المرور. |
| الدخول بحساب غير موجود في `survey_admins` | يفشل تحميل النتائج بسبب سياسات قاعدة البيانات. |
| الدخول بحساب مدير مصرح | تظهر لوحة المؤشرات والجداول والرسوم البيانية. |
| استخدام البحث والفلاتر | تتغير النتائج والإحصاءات حسب البيانات الظاهرة. |
| الضغط على تصدير CSV | يتم تنزيل ملف CSV باللغة العربية. |
| الضغط على طباعة تقرير PDF | تفتح نافذة الطباعة بتنسيق تقرير مناسب. |

## ملاحظات تشغيلية مهمة

يحاول التطبيق منع تكرار الإرسال السريع من نفس الجهاز باستخدام `localStorage` لمدة ست ساعات. هذا الإجراء يقلل التكرار العارض، لكنه لا يعد حماية مطلقة ضد التكرار المتعمد. كما يظهر مؤشر تحميل عند إرسال التقييم، وتظهر رسائل واضحة عند فشل الاتصال أو عند عدم اختيار التقييم العام.

لوحة الإدارة تحسب المتوسطات على البيانات الظاهرة بعد تطبيق البحث والفلاتر. لذلك عند استخدام فلتر نوع الخدمة أو حالة الإنجاز، تتغير الإحصاءات والرسوم البيانية لتعكس النتائج المفلترة فقط. زر تصدير CSV يصدّر النتائج الظاهرة حالياً، وزر الطباعة يستخدم `window.print` لإخراج تقرير منسق يمكن حفظه كملف PDF من نافذة الطباعة.
